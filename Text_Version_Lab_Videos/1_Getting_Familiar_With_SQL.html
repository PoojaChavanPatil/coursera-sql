Getting Familiar with SQL in Jupyter Notebooks

Introduction

Imagine being able to analyze thousands of records with just a few lines of code. That's exactly what data analysts do every day using SQL in Jupyter Notebooks. Today, we'll explore how a bookstore uses these tools to make smart business decisions. By the end of this tutorial, you'll be writing your first SQL queries with confidence.

Learning Objectives

  • Connect to a SQL database in Jupyter Notebook
  • Write and execute basic SELECT statements
  • Explore real business data using SQL queries

Workspace Introduction

First, let's understand our workspace. Jupyter Notebook is like a digital playground where we can write code, see results instantly, and document our process. Each notebook is made up of code cells that can be run individually to show results one block at a time, as well as document our process.

In this course, we'll be using Python code to import SQLite and Pandas. These two tools will enable all of our SQL query magic to happen. SQLite is a sql package and Pandas is a powerful Python library for data manipulation and analysis. We'll get into the deeper details of python and pandas in Course 2, for this course any python code needed will be provided.

import sqlite3

These first few lines are our foundation. Think of them as opening the door to our database. Let me break this down:

  • sqlite3 helps us talk to our database
  • pandas is our tool for displaying data nicely
  • The connect function is like dialing the phone number to our database
  • And cursor? Think of it as our database messenger

First Query Example

Query Concept

Now for the exciting part, your first query! Let's see what books we have in stock. Our query to determine this is Select asterisk from books limit 5.

[Code begins here]

SELECT *
FROM books
LIMIT 5;
[Code ends here]

Query Explanation

This query is like asking, 'Show me everything about the first 5 books.' In the real world, data analysts often start with a small sample to understand their data structure before diving deeper.

Challenge

To view all rows of data from books, you can remove the LIMIT 5 text. To run this new query replace the "YOUR CODE HERE TEXT" in the code cell with our new query: SELECT asterisk FROM books. Feel free to pause your video while you type in the new text.

[Code begins here]

SELECT *
FROM books
[Code ends here]

What do you notice about the data that's returned? We now have all rows of data from the books table. Each row represents a book, and each column tells us something specific about that book.

Specific Columns

In real business scenarios, we often need specific information rather than all data. Let's say BookCycle's manager needs a quick inventory check focusing just on titles. Our new query is SELECT title FROM books

[Code begins here]

SELECT title
FROM books;
[Code ends here]

Asterisk Explanation

Notice how we replaced the asterisk (*) with 'title'. This makes our query:

  • More efficient by retrieving only what we need
  • Easier to read and analyze
  • More focused on our business question

Advanced Query Examples

Now, let's help BookCycle's staff with their inventory check. They need titles, authors, and prices.

[Code begins here]

SELECT title, author, purchase_price
FROM books;
[Code ends here]

Before running this query, what information do you expect to see?

How might this help the bookstore staff?

Think about it

As you may have expected, the output displays all data from the title, author and purchase price columns.

Using WHERE Clause

Data analysis often requires finding specific information. This is a great opportunity to utilize the WHERE clause. The WHERE clause is our filter - think of it as your database's search function.

[Code begins here]

SELECT title, list_price
FROM books
WHERE list_price > 11;
[Code ends here]

Filter Explanation

This query helps BookCycle identify their premium inventory. In retail, this information is crucial for:

  • Pricing strategies
  • Security measures for high-value items
  • Special display arrangements

Common Mistakes

  • Forgetting semicolons at the end of queries
  • Using single quotes instead of double quotes for column names
  • Incorrect spelling of column names

The query would read SELECT title and list_price FROM books WHERE the list_price less than 10.
